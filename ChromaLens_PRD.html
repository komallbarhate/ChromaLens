01 — Background & Problem Statement

1.1 Context

In design, marketing, and product workflows, teams frequently need to understand the color composition of images — for brand audits, packaging reviews, social media consistency, or content moderation. Currently, professionals rely on one of two inadequate approaches:

  • Manual inspection — subjective, time-consuming, and inconsistent across team members.
  • Dominant color palette tools — return raw hex codes (e.g. #4A2C1D) which are meaningless to non-technical users and don't map to human-understandable color names.

1.2 The Gap

There is no lightweight, zero-installation, privacy-preserving tool that:

  • Analyzes images at the pixel level (every single pixel, not just a sample),
  • Maps results to human-readable color categories (e.g. "Navy → Blue", "Burgundy → Red"),
  • Displays results as clear percentage charts, and
  • Works instantly in a browser — no backend, no sign-up, no API cost.

1.3 The Solution

ChromaLens is a browser-based tool that accepts an image, reads every pixel using the HTML5 Canvas API, classifies each pixel into one of 11 universal color categories, and displays an interactive color percentage breakdown as charts, swatches, and an animated summary panel. Additionally, users can tap any point on the image to identify that exact pixel's color in real time.

⚡ Key Differentiator
100% client-side processing — zero pixel data is ever transmitted to any server. The tool works offline once loaded.

02 — Use Case

"As a designer, marketer, or product manager, I want to upload any image and instantly see what percentage of it is each primary color, so I can make informed decisions about visual branding, content, and design."

2.1 Detailed Scenario Walkthrough

Step 1 — Upload

The user opens ChromaLens in their browser. They see a drag-and-drop zone with a "Choose Image" button. They either drag an image file onto the zone or click to open a file picker. Accepted formats: PNG, JPG, WEBP.

Step 2 — Processing

A loading overlay appears ("Analyzing pixels…") with an animated spinner. The tool reads every pixel of the image using the Canvas getImageData API, converts each pixel's RGB values to HSL, and runs it through a 14-rule classification decision tree to assign it to one of 11 color buckets.

Step 3 — Results

Within 1–2 seconds, the results section replaces the upload zone, showing:

  • A preview of the uploaded image (with a crosshair cursor)
  • An Analysis Summary panel with animated color percentage bars
  • A Doughnut chart of color distribution
  • A Horizontal bar chart of percentages by color
  • A Color legend grid with swatches and percentages

Step 4 — Interactive Color Picker

The user taps/clicks any point on the image preview. A floating tooltip pops up (with a spring animation) showing the exact color category (e.g. "Green"), the actual hex value (e.g. #3A8C45), and a colored swatch. A pulsing dot appears at the tap point.

Step 5 — Export or Reset

The user can download the doughnut or bar chart as a PNG file, or click "Analyze Another" to return to the upload state.

07 — Color-Mapping Algorithm

📐 Developer Note
This section is the authoritative specification. A developer should be able to implement an identical classification engine from this section alone without referring to any other source.

7.1 Pipeline Overview

Raw Image File (PNG / JPG / WEBP)
↓
Stage 1: Draw to hidden canvas (downscale to max 400px)
↓
Stage 2: Read ImageData — iterate every pixel (R, G, B, A)
↓
Stage 3: RGB → HSL Conversion
↓
Stage 4: Decision Tree → assign to 1 of 11 buckets
↓
Stage 5: Count pixels → compute percentages
↓
Charts, Summary Panel, Swatch Legend

7.2 Stage 1 — Image Downscaling

For performance, the image is drawn onto a hidden <canvas> downscaled so that the longest dimension does not exceed 400px, preserving aspect ratio.

scale        = min(1,  400 / max(naturalWidth, naturalHeight))
canvasWidth  = round(naturalWidth  × scale)
canvasHeight = round(naturalHeight × scale)
ctx.drawImage(img, 0, 0, canvasWidth, canvasHeight)
💡 Why 400px?
This limits pixel count to ≤ 160,000, keeping analysis under 100ms on most devices while maintaining statistical accuracy. Color distribution is not materially affected by this downscale.

7.3 Stage 2 — Pixel Iteration

imageData = ctx.getImageData(0, 0, canvasWidth, canvasHeight)
data = imageData.data   // Uint8ClampedArray — RGBA interleaved

for i = 0 to data.length step 4:
    R = data[i]       // Red   0–255
    G = data[i+1]     // Green 0–255
    B = data[i+2]     // Blue  0–255
    A = data[i+3]     // Alpha 0–255

    if A < 20: continue    // skip fully transparent pixels

    label = classifyPixel(R, G, B)
    counts[label]++

7.4 Stage 3 — RGB to HSL Conversion

All pixel values are converted from RGB (0–255) to HSL — Hue (0–360°), Saturation (0–100%), Lightness (0–100%) — using the standard conversion formula:

r' = R / 255,   g' = G / 255,   b' = B / 255

cmax = max(r', g', b')
cmin = min(r', g', b')
Δ    = cmax - cmin

── LIGHTNESS ─────────────────────────────────
L = (cmax + cmin) / 2

── SATURATION ────────────────────────────────
if Δ == 0:        S = 0   (achromatic)
else if L < 0.5:  S = Δ / (cmax + cmin)
else:             S = Δ / (2 - cmax - cmin)

── HUE ───────────────────────────────────────
if Δ == 0:          H = 0  (achromatic)
else if cmax == r': H = ((g' - b') / Δ) mod 6  × 60
else if cmax == g': H = ((b' - r') / Δ + 2)    × 60
else if cmax == b': H = ((r' - g') / Δ + 4)    × 60

if H < 0: H += 360

Final output: L = L×100,  S = S×100  (both in range 0–100)
💡 Why HSL?
HSL separates color (Hue) from vividness (Saturation) and brightness (Lightness), mirroring human color perception. This makes achromatic detection and hue-wheel classification natural and accurate.

7.5 Stage 4 — Classification Decision Tree

The following 14 rules are evaluated top-to-bottom. The first matching rule wins. All range endpoints are inclusive unless noted with ( or ).

Rule 1 — WHITE (Achromatic)
if L ≥ 88 → return "White"

Catches: ivory, cream, off-white, snow, eggshell

Rule 2 — BLACK (Achromatic)
if L ≤ 12 → return "Black"

Catches: jet black, very dark charcoal, near-black navy

Rule 3 — GRAY (Achromatic)
if S < 16 → return "Gray"

Catches: silver, ash, slate, all near-neutral mid-tones regardless of hue

Rule 4 — BROWN (Primary)
if H ∈ [0, 40] AND S ∈ [15, 60] AND L ∈ [15, 46] → return "Brown"

Warm hue + moderate saturation + dark-ish = earth tone. Catches: tan, sienna, chocolate, saddle brown

Rule 5 — BROWN (Orange-brown extension)
if H ∈ (10, 44] AND S ∈ [15, 55] AND L ∈ (22, 44] → return "Brown"

Catches: terracotta, rust tones, warm orange-browns Rule 4 misses

Rule 6 — PINK (High-lightness reddish)
if (H ≥ 315 OR H ≤ 14) AND L ≥ 60 AND S ≥ 15 → return "Pink"

Reddish hue too light to be Red. Catches: rose, hot pink, salmon, coral

Rule 7 — PINK (Desaturated blush)
if (H ≥ 315 OR H ≤ 14) AND S < 50 AND L ≥ 46 → return "Pink"

Low-saturation reddish-light. Catches: blush, pastel pink, light skin tones

Rule 8 — RED (Hue Wheel)
if H ≤ 14 OR H ≥ 345 → return "Red"

Catches: scarlet, crimson, burgundy, maroon, fire red, dark red

Rule 9 — ORANGE (Hue Wheel)
if H ∈ (14, 44] → return "Orange"

Catches: amber, tangerine, pumpkin, fire orange

Rule 10 — YELLOW (Hue Wheel)
if H ∈ (44, 72] → return "Yellow"

Catches: gold, lemon, mustard, khaki, light yellow

Rule 11 — GREEN (Hue Wheel)
if H ∈ (72, 165] → return "Green"

Catches: lime, olive, forest, emerald, teal, sage, cyan-green

Rule 12 — BLUE (Hue Wheel)
if H ∈ (165, 265] → return "Blue"

Catches: navy, sky blue, cobalt, indigo, cerulean, cyan-blue

Rule 13 — PURPLE (Hue Wheel)
if H ∈ (265, 314] → return "Purple"

Catches: violet, lavender, mauve, plum, magenta-purple

Rule 14 — PINK (Hue Wheel fallback)
if H ∈ (314, 360) → return "Pink"

Catches: remaining pinkish-magenta hues not caught by Rules 6–7

7.7 Stage 5 — Percentage Calculation

totalPixels = count of pixels where alpha ≥ 20

for each category C:
    percentage(C) = counts[C] / totalPixels × 100

Results sorted descending by pixel count before display.
Sum of all percentages ≈ 100% (±0.1% due to floating-point rounding).

7.8 Color Picker — Pixel Coordinate Mapping

When the user clicks the displayed image (rendered with object-fit: contain), the click must be mapped to canvas coordinates:

// Step 1: Get element dimensions
elementW = img.clientWidth
elementH = img.clientHeight
naturalW = img.naturalWidth
naturalH = img.naturalHeight

// Step 2: Compute contained image rect (accounting for letterboxing)
scale     = min(elementW / naturalW, elementH / naturalH)
renderedW = naturalW × scale
renderedH = naturalH × scale
offsetX   = (elementW - renderedW) / 2
offsetY   = (elementH - renderedH) / 2

// Step 3: Map click to image-relative coordinates
relX = clickX - offsetX
relY = clickY - offsetY
if relX < 0 or relY < 0 or relX > renderedW or relY > renderedH: ignore

// Step 4: Map to canvas pixel
canvasX = clamp(round(relX / renderedW  × canvasWidth),  0, canvasWidth-1)
canvasY = clamp(round(relY / renderedH × canvasHeight), 0, canvasHeight-1)

// Step 5: Read and classify
[R,G,B,A] = ctx.getImageData(canvasX, canvasY, 1, 1).data
if A < 20: ignore
colorName = classifyPixel(R, G, B)
hex = "#" + R.hex + G.hex + B.hex  (uppercase, zero-padded)

15 — Appendix: Worked Examples

Example A — Dark Red RGB(139, 0, 0)

Step 1 — RGB → HSL:
  r' = 0.545,  g' = 0,  b' = 0
  cmax = 0.545, cmin = 0, Δ = 0.545
  L = 27.3%,  S = 100%,  H = 0°

Step 2 — Decision Tree:
  Rule 1: L=27.3% < 88 → skip
  Rule 2: L=27.3% > 12 → skip
  Rule 3: S=100% ≥ 16 → skip
  Rule 4: S=100% > 60 → skip (out of [15,60] range)
  Rule 6: L=27.3% < 60 → skip
  Rule 8: H=0° ≤ 14 → ✅ RED

Result: RGB(139,0,0) → RED  ✓  (Dark Red correctly maps to Red)

Example B — Navy Blue RGB(0, 0, 128)

Step 1 — RGB → HSL:
  r' = 0, g' = 0, b' = 0.502
  L = 25.1%,  S = 100%,  H = 240°

Step 2 — Decision Tree:
  Rules 1–7: Not matched (H=240°, L=25.1%, S=100%)
  Rule 12: H=240° ∈ (165, 265] → ✅ BLUE

Result: RGB(0,0,128) → BLUE  ✓  (Navy correctly maps to Blue)

Example C — Light Gray RGB(180, 180, 180)

Step 1 — RGB → HSL:
  r' = g' = b' = 0.706  →  Δ = 0 (achromatic)
  L = 70.6%,  S = 0%,  H = 0°

Step 2 — Decision Tree:
  Rule 1: L=70.6% < 88 → skip
  Rule 2: L=70.6% > 12 → skip
  Rule 3: S=0% < 16 → ✅ GRAY

Result: RGB(180,180,180) → GRAY  ✓

Example D — Burgundy RGB(128, 0, 32)

Step 1 — RGB → HSL:
  r' = 0.502, g' = 0, b' = 0.125
  cmax = 0.502 (r'), cmin = 0, Δ = 0.502
  L = 25.1%,  S = 100%
  H = ((0 - 0.125) / 0.502 + 0) × 60 = (-0.249 + 6) × 60 = 345.2°

Step 2 — Decision Tree:
  Rules 1–7: Not matched
  Rule 8: H=345.2° ≥ 345 → ✅ RED

Result: RGB(128,0,32) → RED  ✓  (Burgundy correctly maps to Red)
